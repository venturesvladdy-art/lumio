# SkillSprinter

**Master any skill, one question at a time.** An AI-personalized, gamified
micro-learning platform. Pick a skill, answer a few quick questions, and an
agent builds a custom plan of up to 100 questions that adapts as you learn —
in **English or Polish**.

> This is a **clickable design prototype**. The UI, navigation, gamification,
> i18n and the full learning flow are real and runnable with **zero API keys**.
> The AI agent, payments, accounts and database are mocked behind clean, clearly
> documented seams so they can be swapped for real services without touching the
> rest of the app.

---

## Quick start

```bash
npm install
npm run dev
# open http://localhost:3000
```

Production build:

```bash
npm run build
npm run start
```

Requires Node 18+ (built and tested on Node 24).

## What's in the prototype

| Page | Route | What it does |
|------|-------|--------------|
| Landing | `/` | Hero, how-it-works, features, gamification, skills & pricing teasers |
| Skills | `/skills` | 5 predefined tracks (SAT, GMAT, Interpersonal, Engineering, AI) + custom "Other" |
| Onboarding | `/learn/[skill]` | 5–10 adaptive questions → animated agent → personalized plan |
| Session | `/learn/[skill]/session` | One-question-at-a-time Q&A, instant feedback, XP, combos |
| Dashboard | `/dashboard` | Level, XP, streak, daily goal, badges, per-skill progress, plan usage *(requires sign-in)* |
| Pricing | `/pricing` | Basic / Smart / Guru with monthly–yearly toggle and FAQ |
| Login | `/login` | Email + simulated Google / Apple sign-in |
| Contact | `/contact` | Validated contact form → `/api/contact` |
| Privacy / Terms | `/privacy`, `/terms` | Draft policy & terms (placeholder legal copy) |

**Gamification:** XP per answer, 10 levels (Novice → Luminary), daily streaks,
8 badges, combo bonuses, daily-goal rings.

**Plans & gating (enforced in the UI):**

| Plan | Price | Questions/day | Skills | Tracking |
|------|-------|---------------|--------|----------|
| Basic | Free | 5 | 3 | — |
| Smart | $10/mo · $100/yr | 20 | Unlimited | ✓ |
| Guru | $40/mo · $350/yr | Unlimited | Unlimited | ✓ (advanced) |

**i18n:** Full English ⇄ Polish switch (top-right toggle). All UI chrome and the
question bank are bilingual. Language preference persists in `localStorage`.

**Sign in:** Email plus simulated *Continue with Google / Apple* (prototype — no
real OAuth round-trip; structured for a one-file swap to Auth.js). The dashboard
and learning flow require sign-in. Accounts in the allowlist in
[`lib/auth.tsx`](lib/auth.tsx) (`GURU_EMAILS`) are auto-granted the **Guru** plan
on sign-in — `vladimir.s.anokhin@gmail.com` is pre-added.

**Real Claude:** Plans are generated by Claude when `ANTHROPIC_API_KEY` is set
(see **Enabling Claude** below); otherwise the curated bank is used. Either way
the flow is identical.

## Project structure

```
app/                     # Next.js App Router pages (route per folder)
  layout.tsx             # fonts (Inter + Sora), providers, nav, footer
  page.tsx               # landing
  skills/ pricing/ dashboard/
  learn/[skill]/         # onboarding + plan reveal
  learn/[skill]/session/ # the learning session

  i18n/                  # dictionary.ts (EN/PL) + provider/hooks
  types.ts               # shared types
  skills.ts              # skill catalog + custom-skill builder
  content.ts             # bilingual Q&A bank
  onboarding.ts          # questionnaire + level derivation
  agent.ts               # plan assembly (shared by Claude + bank paths)
  planClient.ts          # client → /api/generate-plan (with local fallback)
  auth.tsx               # sign-in context + Guru allowlist + route guard
  gamification.ts        # levels, badges, XP/combo logic
  plans.ts               # Basic/Smart/Guru config + gating helpers
  store.tsx              # app state (React context + localStorage)
app/api/
  generate-plan/         # ⭐ real Claude call (claude-opus-4-8) + bank fallback
  contact/               # contact-form handler
```

State lives in `localStorage` (no backend). Clear it via the **reset** button on
the dashboard, or `localStorage.clear()` in devtools.

## Enabling Claude

Plan generation already calls the **real Claude API** — it just needs a key:

```bash
cp .env.local.example .env.local
# edit .env.local and set ANTHROPIC_API_KEY=sk-ant-...
npm run dev
```

With a key, [`app/api/generate-plan/route.ts`](app/api/generate-plan/route.ts)
asks `claude-opus-4-8` for personalized, bilingual questions (structured JSON
output, key kept server-side). Without a key it falls back to the curated bank,
so nothing breaks. The model is chosen by tier (Sonnet for free, Opus for
smart/guru); override with `SKILLSPRINTER_MODEL` to force one everywhere.

## The real backend (Auth + DB + Stripe)

A full backend is **built and type-checked**, behind one flag
(`NEXT_PUBLIC_USE_DB`). Default is the zero-setup demo; set the flag + connect
accounts to switch on:

- **Auth.js (NextAuth v5)** — email/password + Google sign-in ([`auth.ts`](auth.ts)).
- **Neon Postgres + Prisma** — full **audit log**: every curriculum, Q&A and
  answer stored per user/skill ([`prisma/schema.prisma`](prisma/schema.prisma)).
- **Stripe** — Checkout, Customer Portal, and a webhook that syncs the plan
  ([`app/api/stripe`](app/api/stripe)).

👉 **Follow [SETUP.md](SETUP.md)** to provision Neon, Google OAuth, and Stripe,
run the migration, and flip the flag. (Apple sign-in was skipped — it needs a
$99/yr Apple Developer account; add it later as another Auth.js provider.)

What still needs your accounts/keys before it can run: a Neon database, an
`AUTH_SECRET`, Google OAuth credentials, and Stripe keys/price IDs. Until then
the app keeps running in demo mode.

**Scale (10k concurrent → 1M users).** The app is stateless; deploy on Vercel.
Cache plan generation by `(skill, level, focus)` profile (Redis/KV), and cache
`User.tier` instead of reading it per request. Nothing here holds server state.
